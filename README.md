# DNS Proxy Server

## Назначение
Этот код представляет собой простой DNS прокси-сервер, который:
1. Принимает DNS-запросы от клиентов
2. Проверяет наличие ответа в кеше
3. Если ответа нет в кеше - перенаправляет запрос на вышестоящий DNS-сервер
4. Кеширует полученные ответы для последующего использования
5. Возвращает ответ клиенту

## Основные компоненты системы

### 1. Класс Cache
Функционал:
- Автоматическая загрузка кеша из файла при старте (load_file)
- Сохранение кеша в файл при завершении (write_file)
- Добавление новых записей (append)
- Проверка наличия записей (__contains__)
- Автоматическая очистка просроченных записей

### 2. Класс DnsParse
Функционал:
- Парсинг входящих DNS-пакетов
- Извлечение:
  - Транзакционного ID
  - Флагов
  - Счетчиков записей
- Чтение секций ответа:
  - Answers
  - Authority
  - Additional
- Формирование ответного пакета из кеша (complete_to_full_packet)

### 3. Вспомогательные функции

| Функция          | Назначение                                                                 |
|------------------|---------------------------------------------------------------------------|
| url_to_bytes   | Преобразует доменное имя в DNS-формат                                    |
| read_name_2    | Чтение доменного имени из пакета (с поддержкой сжатия)                  |
| read_query     | Извлечение информации о запросе из пакета                               |
| get_info       | Отправка запроса на вышестоящий DNS-сервер                             |

### Запуск сервера
Требования:

- Python 3.x
- Стандартные библиотеки (socket, struct, pickle)

Команда запуска:

bash
python dns_proxy.py

Кеш автоматически сохраняется в файл dns_cache.pickle при завершении работы.

Поддерживаемые типы DNS-записей

| Тип      |  Описание                |
|------------------|-----------------|
| A       | IPv4 адрес               |
| NS      | DNS сервер               |
| SOA     | Start of Authority       |
| AAAA    | IPv6 адрес               |

### Настройки сервера

# Конфигурация сервера
PORT = 53
IP = '127.0.0.1'
TIMEOUT = 1  # секунды

# Внешний DNS-сервер
UPSTREAM_DNS = ('212.193.163.7', 53)
Примеры работы
Сценарий 1: Первый запрос

- Клиент запрашивает example.com
- Прокси проверяет кеш - записи нет
- Запрос перенаправляется на 212.193.163.7
- Ответ кешируется
- Клиент получает ответ

Сценарий 2: Повторный запрос

- Клиент запрашивает example.com
- Прокси находит запись в кеше
- Проверяется TTL записи
- Если TTL не истек - клиент получает ответ из кеша
